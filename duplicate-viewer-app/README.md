# Photo Duplicate Viewer

Interactive web application for reviewing and deleting photo duplicates found by the photo-organizer script.

## Features

- 📸 Side-by-side duplicate comparison
- 🗑️ **Real-time file deletion** - deletes happen immediately when you click Delete
- ⌨️ Keyboard shortcuts for fast review
- 📊 Metadata display (resolution, file size, dates)
- 💾 Export decisions log
- ✅ Visual feedback for keep/delete decisions

## Quick Start

### 1. Generate duplicates.csv

First, run the photo organizer to find duplicates:

```bash
cd "C:\dscodingpython\File organizers"
photoenv\Scripts\activate
python photo-organizer.py --root "C:\path\to\your\photos" --dry-run
```

This creates `duplicates.csv` in the File organizers folder.

### 2. Install Dependencies

**Backend (Python):**
```bash
cd duplicate-viewer-app\backend
pip install -r requirements.txt
```

**Frontend (Node.js):**
```bash
cd duplicate-viewer-app
npm install
```

### 3. Run the Application

**Option A: Use the startup script (Windows)**
```bash
start.bat
```

**Option B: Manual start**

Terminal 1 - Backend:
```bash
cd backend
python server.py
```

Terminal 2 - Frontend:
```bash
npm run dev
```

Then open http://localhost:3000

## How It Works

1. **Load Duplicates**: The app loads from `duplicates.csv` generated by photo-organizer.py
2. **Review**: Navigate through duplicate pairs with arrow keys or buttons
3. **Decide**: Click "Keep" or "Delete" for each image
   - **Delete is immediate**: Files are deleted from your filesystem right away
   - Keep marks the file to preserve
4. **Export**: Save your decisions log as JSON

## Keyboard Shortcuts

- `←` `→` - Navigate between duplicate sets
- `1` - Keep original (left image)
- `2` - Keep duplicate (right image)

## API Endpoints

The Flask backend provides:

- `GET /api/duplicates` - Load all duplicates with metadata
- `POST /api/delete` - Delete a file immediately
- `POST /api/file-exists` - Check if file still exists
- `POST /api/export-decisions` - Save decisions log

## Safety Features

- Deletion only allowed in specific directories (configurable in backend/server.py)
- Confirmation before exporting incomplete reviews
- Visual indicators for deleted files
- Decision log export for record-keeping

## Configuration

Edit `backend/server.py` to configure:

- `PHOTO_ROOT` - Base directory for photos
- `DUPLICATES_CSV` - Path to duplicates CSV file
- Allowed deletion directories (safety check)

## Troubleshooting

**"Cannot load duplicates"**
- Ensure `duplicates.csv` exists in `File organizers` folder
- Check Flask server is running on port 5000

**Images not displaying**
- Browser security may block `file://` protocol
- Images must be accessible from your local filesystem

**Deletions not working**
- Check file paths are within allowed directories
- Verify file permissions
